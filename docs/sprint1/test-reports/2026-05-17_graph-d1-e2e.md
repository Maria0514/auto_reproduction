## 1. 任务背景与设计要点

D1（`core/graph.py` LangGraph 主图骨架）由全栈开发代理交付，含 7 节点注册（paper_intake / paper_analysis 为 ReAct wrapper + 5 个 pass-through 占位）、START→...→END 顺序边、`build_graph(checkpointer=None)` 工厂、未传 checkpointer 时懒导入 `core.checkpointer.get_checkpointer()`。`tests/test_graph.py` 13 项单测验证了图结构合法性、占位返回值、wrapper 同一性、mock checkpointer 编译、patch ReAct wrapper 后的 invoke 链路连通。

本轮 e2e 聚焦 graph 集成视角的**新增**维度，避免和现有 `tests/test_paper_intake_e2e.py`（4 用例）/ `tests/test_paper_analysis_e2e.py`（6 用例）做无谓重复——后两者已分别验证节点本体在真实 LLM + 真实 deepxiv SDK 下的契约。

**新增覆盖**：
1. 节点串接：paper_intake 写入的 `paper_meta` 经顺序边流入 paper_analysis 节点（验证 GlobalState 合并语义在 graph 中符合 PRD 预期）。
2. 占位节点透明性：5 个占位节点在 invoke 整条链路时不会丢弃或覆盖前两个 ReAct 节点的产出。
3. SqliteSaver 持久化：真实 SQLite 文件作为 checkpointer，断言 invoke 后文件被写入，新实例 `get_state(config)` 能完整读回 paper_meta + paper_analysis。
4. PRD §6.1 AC-1 + AC-2 全步骤逐条断言。

**主路径论文**：`arXiv:2405.14831` (HippoRAG)。沿用 paper_intake / paper_analysis e2e 已经验证可访问 + 章节齐全 + CS 领域的同一篇靶论文，减少跨用例论文差异，省 token 与时间。

**SQLite 文件隔离**：用 `tmp_path_factory.mktemp("graph_e2e_checkpoints")` 提供 module 级独立目录，主流水线 invoke 完毕后显式关闭连接强制 WAL 刷盘；后续用新 `SqliteSaver` 实例 reopen 同一文件做回读，模拟 PRD AC-2 第 4 步"新建 graph 实例使用相同 thread_id 可恢复"的语义。

## 5. 实现取舍 / 已知限制

### 5.1 TC-E2E-D1-04 保留位未启用

异常 arXiv ID（如不存在论文）路径在 graph 集成视角下的断言已在 paper_intake e2e（`test_e2e_node_errors_empty_on_success` 反向覆盖）+ paper_analysis 单测（`tests/test_paper_analysis.py` CP3 paper_meta=None 前置校验）层级覆盖；graph 层只是顺序边把 NodeError 传播下去，**不引入新增逻辑**。本轮为控制 e2e 总时长 < 10 分钟 + 防止罕见的 deepxiv "论文不存在" 接口抖动引入 flakiness，暂不开启该用例。

后续触发条件：（a）graph.py 引入条件路由（Sprint 3 修复循环）后异常路径不再"透传"；（b）回归发现 NodeError 在 graph 层被吞掉。届时启用 TC-04 即可，文件末尾已留位。

### 5.2 SystemMessage Prompt Cache 字节级断言不在本套件

已由 `tests/test_paper_analysis_e2e.py::test_e2e_prompt_cache_system_prompt_byte_identical` 覆盖（双论文真实 LLM 链路 + ChatOpenAI.invoke monkeypatch 抓取 SystemMessage），本套件不重复。

### 5.3 LLM 抖动观察

3 次连跑均未触发 BUG-S1-03 类 sections_read 漏写场景；BUG-S1-03 已通过 `_backfill_analysis_from_tools` 修复，本套件作为 graph 层的回归同样未观察到漏写。HippoRAG 章节结构规范，是工具调用兜底链最稳的场景之一；若后续切换非规范命名论文做 graph e2e，需要重新评估稳定性。

### 5.4 时间预算

单套件 3 用例耗时 60-80 秒（fixture share 后只跑一次主流水线）；全量含 16 个 e2e 用例耗时 ~4 分钟。距离测试工程师代理"整套别超 10 分钟"的目标还有充裕空间，TC-04 启用余地仍在。
