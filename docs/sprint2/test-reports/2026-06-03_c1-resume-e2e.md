### 省 token / 防抖设计
- "跑到 planning 暂停" 是全套最贵环节（≈ 一次 paper_analysis 全链路真实 LLM + deepxiv）。
  设计为两条 thread 各跑一次（approve_thread / cancel_thread，module-scope fixture 缓存）：
  - approve thread 单 thread 内同时验证 revise(self-loop) + approve(next) + SqliteSaver 跨实例回读；
  - cancel thread 独立隔离（thread 状态按 thread_id 持久化，cancel 把 thread 推进到 END，故必须
    与 approve thread 分离）。
- revise/approve/cancel 的 resume 步骤本身不再触发上游真节点（已 checkpoint），仅 planning
  内部 ReAct 子图在 revise self-loop 时再跑一次（成本可控）。
